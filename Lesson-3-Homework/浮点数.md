# 0.1+0.2！=0.3的原因

javascript中浮点数采用IEEE754编码，其中0.1,0.2,0.3在IEEE754中分别表示为

| IEEE754                                                      | 十进制 |
| ------------------------------------------------------------ | ------ |
| 0b0011111110111001100110011001100110011001100110011001100110011010 | 0.1    |
| 0b0011111111001001100110011001100110011001100110011001100110011010 | 0.2    |
| 0b0011111111010011001100110011001100110011001100110011001100110011 | 0.3    |

对IEEE754下的0.1和0.2进行二进制相加，所得结果为0b0011111111010011001100110011001100110011001100110011001100110100，即十进制的0.30000000000000004，而不是0.3。

在编程中，通常会采用取插值并和一个很小的阈值比较的方法来判断两个浮点数是否相等。例如以下代码：

```python
a=0.1
b=0.2
c=a+b
if(abs(c-0.3)<0.0000001)then
{...}
```
